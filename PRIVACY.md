# Privacy

This notice describes the privacy boundary of the AMT-01 Open Investigation at its initial public launch.

## The Project Website

The launch website is a static site. At launch, the project does not operate user accounts, submission forms, private messaging, mailing lists, analytics, advertising trackers, or project-set cookies through the website.

The hosting provider may still process technical data needed to deliver and protect the site, such as an IP address, request time, requested URL, browser or device information, and security logs. That processing is controlled by the hosting provider and its own privacy terms, not by a project-managed database.

## GitHub and Discord

The project uses GitHub for the public repository and Discord for public community discussion. GitHub and Discord operate their own accounts, infrastructure, logs, cookies, moderation systems, and privacy practices. Review their terms and privacy notices before using those services.

GitHub issues, pull requests, commits, review comments, and other public repository activity are public. Discord channels identified as public should also be treated as public. Public content may be indexed, quoted, copied, archived, mirrored, or included in repository history.

A private direct message to a Discord moderator is not visible in a public channel, but it is still processed by Discord. Use moderator direct messages only for conduct, privacy, takedown, or security concerns, and share the minimum information necessary.

## Do Not Post Sensitive Material

Do not post raw witness reports or identifying information in public GitHub or Discord spaces. This includes:

- names and private contact details;
- exact private locations or details that reveal a person's home, workplace, or lodging;
- faces, voices, vehicle identifiers, or personal documents;
- original files containing location, device, account, or other sensitive metadata;
- medical, employment, immigration, financial, or similarly sensitive information;
- information about another person that you do not have permission to share.

Before making a public contribution, remove information that is not necessary for the research purpose and check files for hidden metadata. A public username, profile, commit author name, or commit email may also reveal your identity.

## No Private Experience Intake Yet

The project does not currently accept private experience or witness-report submissions. Do not send raw accounts, sensitive media, or identifying evidence to moderators or contributors by direct message. A future intake route will open only after consent, storage, access, retention, redaction, deletion, and takedown procedures are operational and clearly published.

Public discussion about already published material is welcome within the community rules. It is not a substitute for a private intake process.

## Repository History and Copies

Material intentionally submitted to a public Git repository can remain in commit history, forks, clones, mirrors, caches, release archives, or copies held by other people. The project can correct or remove material from locations it controls, but it cannot guarantee deletion from independent copies.

## Privacy Concerns

For a privacy concern about material controlled by the project, send a private direct message to a moderator in the [project Discord server](https://discord.gg/JVvHf5cXhs). Do not post sensitive details publicly. State which project-controlled page, file, message, or asset is involved and what action you are requesting. The project may ask for enough information to assess the request while avoiding unnecessary collection.

Requests are handled under [CONSENT_AND_TAKEDOWN.md](CONSENT_AND_TAKEDOWN.md). Platform-level requests concerning GitHub, Discord, or the hosting provider may also need to be made directly to that provider.

## Changes to This Notice

This notice will be updated before the project adds any project-controlled account, form, analytics system, cookie, or private experience intake. Material changes will be documented in the repository.
