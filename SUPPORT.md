# Support and contact

## Public questions

Use the [AMT-01 Discord community](https://discord.gg/JVvHf5cXhs) for:

- questions about the project or documentation;
- help finding the relevant file or contribution route;
- public discussion of an explanation or source;
- website feedback that contains no sensitive information;
- community orientation.

Keep public questions focused on already published material. The community and repository are maintained by volunteers, so no response time is guaranteed.

## Repository work

Use the appropriate public GitHub issue or pull-request route for reproducible bugs, documentation corrections, calculation audits, source corrections, and contribution proposals.

Do not open a public issue for sensitive conduct, security, identity, or witness-report material.

## Sensitive conduct or security matters

For harassment, threats, doxing, impersonation, moderation concerns, credentials, or a security issue that should not be public, privately direct-message a server moderator on Discord.

In the first message, share only the minimum needed to route the concern. Do not include credentials, exploit details, identity documents, exact private locations, or sensitive media until a moderator confirms a safe next step.

If the concern involves a moderator, privately contact a different moderator or the project steward.

## Private witness intake

The [AMT-01 witness-intake form](https://tally.so/r/5BgZMb) is the project's only private route for a new experience report and its primary route for access, copy, correction, consent-withdrawal, deletion, or another privacy request concerning an earlier private submission. Privacy requests may also be sent to [werbitsky@gmail.com](mailto:werbitsky@gmail.com). Tally hosts the form, and project-level access to raw responses is limited to the project steward acting as the private-intake curator.

Name, pseudonym, ordinary follow-up contact, and file uploads are optional for a new report. A rights request requires a response email so the curator can verify the requester and communicate the outcome. Uploads are limited to five files of 10 MB each and restricted to images, video, audio, PDF, and plain text. Use only a general location, remove unnecessary metadata, and read [PRIVACY.md](PRIVACY.md) and [CONSENT_AND_TAKEDOWN.md](CONSENT_AND_TAKEDOWN.md) before submitting. A submission is private by default and is not automatically published or accepted as a case record.

Never post or send through public project channels:

- a raw witness report containing personal information;
- a name, address, phone number, private email, or identity document;
- an exact private location or travel record;
- faces, voices, vehicle registrations, or identifying background details;
- original files containing GPS, device, account, or location metadata;
- material involving a minor or vulnerable person;
- medical or mental-health information;
- credentials, access tokens, private keys, or unreleased vulnerability details;
- third-party media you do not have permission to share.

A direct message to a moderator is a route for sensitive conduct or security triage. It is not a witness-intake channel; use the Tally form for that purpose.

## What to include in a public help request

When safe to post publicly, include:

- a short description of the problem or question;
- the relevant public file, section, claim ID, decision ID, or URL;
- steps to reproduce a website or validation problem;
- expected and actual behavior;
- non-sensitive environment details when technically relevant;
- what you already tried.

Do not attach private logs or screenshots without checking them for names, file paths, tokens, coordinates, and other identifying data.

## Scope of support

Project discussion is not legal, medical, mental-health, emergency, aviation-safety, law-enforcement, or security-response advice. If there is an immediate risk to a person, contact the appropriate local emergency or professional service instead of waiting for a project response.
