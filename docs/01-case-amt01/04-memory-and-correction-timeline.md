# AMT-01 Memory and Correction Timeline

This record preserves how the documented recollection has changed. A later amendment does not create new event evidence and is not automatically more accurate. It makes the current witness record explicit while keeping earlier project values auditable.

## Known timeline

| Date or period | Record event | What is known |
|---|---|---|
| Summer 2004 | Recalled observation | The witness currently places the event in summer 2004 near the Antalya coast. The exact day, month, and site remain unknown. |
| Unknown | First written or recorded account | The project has not yet established the date, form, wording, or preservation status of the earliest account. |
| Before 2026-07-19 | Earlier project synthesis | The v1.0 and supplied v1.1 records encoded the event year as unknown, lateral distance as roughly 10–15 m, and black-region count as roughly 5–10. |
| 2026-07-19 | v1.1 knowledge-base package | A modular retrospective synthesis and ADR-style decision narrative were recorded. This is not a contemporaneous event record. |

| 2026-07-20 | Later underside reconstruction supplied | A new reconstruction image was selected for the public website. It is not event media, measured geometry, or an original event-era sketch. |

## Record rules

1. Never silently overwrite a previous value.
2. Keep the prior wording, source namespace, and version available through a typed amendment or supersession link.
3. Distinguish event date, report date, amendment date, and document-recording date.
4. Describe confidence as memory stability, salience, or estimate basis, not as truth probability.
5. Do not infer missing chronology, wording, or tool history.
6. A witness amendment may change downstream calculations without changing the evidential tier of the case.
