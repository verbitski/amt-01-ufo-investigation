# Change Log

## v1.2 public derivative: 2026-07-20

- Created a sanitized public derivative under the neutral identity **AMT-01 Open Investigation**.
- Added a plain-language evidence boundary: one retrospective witness record, no known event media, no independent witness account, no sensor confirmation, no recovered exact site, and no instrumented geometry.
- Recorded later witness amendments additively: summer 2004 with exact day and month unknown, approximately 15–20 m closest lateral distance, and approximately 5–7 completely black regions.
- Added a memory and correction timeline so earlier values remain visible and no amendment is mistaken for new event evidence.
- Published permanent source-qualified mappings for the conflicting v1.0 decision identifiers and added full public-release ADRs `DEC-027` through `DEC-031`.
- Requalified engineering calculations as conditional on retrospective inputs and separated completely black appearance from the later region-count estimate.
- Reframed negative observations as signatures not noticed or detected under recalled conditions, rather than measured physical zeros.
- Kept perceptual interpretation, memory reconstruction, scale error, and conventional identification open for testing.
- Separated Force Skin, MEVE, Q-MEVE, high-speed, transmedium, warp-related, and control material into `docs/04-speculative-models/` with an explicit non-evidence boundary.
- Added public documentation navigation, canonicality rules, a public manifest, and immutable source-package identifiers without publishing private local paths.
- Included a partial machine-readable migration fixture and schema prototype; neither replaces the canonical witness record.
- Intentionally omitted `MASTER_DOSSIER.md` from the public derivative until a deterministic generator and drift check are committed.
- Added a mixed-license release boundary: Apache-2.0 for software and schemas, CC BY 4.0 for project-created documentation, structured public data, the witness record, and rights-controlled reconstruction media, with attribution and a canonical-source backlink.
- Excluded third-party works and local launch material from the project license grants.
- Moved unlabeled and superseded reconstruction files outside public release and deployment paths.
- Added a deterministic FFmpeg media build, baked **RECONSTRUCTION / NOT EVENT MEDIA** into every public image, and verified that publication-master pixels outside the disclosure rectangle remain unchanged.
- Added a machine-readable provenance manifest for all six public reconstruction families and eighteen responsive derivatives, including hashes, dimensions, source-master identifiers, transformation parameters, rights status, and explicitly unknown production details.
- Added an automated media-provenance validator so image inventory, disclosure status, byte sizes, and hashes cannot drift silently.
- Added launch governance, contribution, conduct, AI-use, privacy, consent, takedown, security, and support policies with explicit witness-curator and public/private boundaries.
- Published a consent-based private Tally witness-intake form under `DEC-035`, identified the individual operator through GitHub `@verbitski`, and added `werbitsky@gmail.com` as the durable private contact. Public website deployment remains gated on readable public policies and operational security checks. GitHub, public Discord, ordinary email, and unsolicited moderator messages remain closed to raw reports.
- Documented curator-only project access; optional identity and ordinary follow-up fields; a required response email for privacy requests; optional uploads restricted to five images, video, audio, PDF, or plain-text files of 10 MB each; location and metadata minimization; separate publication permission; manual deletion of every raw Tally submission from the active response list within 12 months; Tally's provider-controlled Trash and backup lifecycle; reCAPTCHA processing; and a 30-day response or initial-assessment target for privacy requests.
- Adopted DCO 1.1 sign-off and path-based inbound-equals-outbound contributions instead of a contributor license agreement.
- Added guarded GitHub issue forms, a pull-request checklist, link validation, automated repository checks, and pull-request DCO enforcement.
